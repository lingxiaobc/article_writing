# 小红书信息图生成技能

## 技能定位

本技能专为**小红书内容创作**场景设计，核心功能是：

- 读取参考文章，按二级标题（`##`）分割为多个板块
- 对每个板块内容进行提炼与总结
- 利用子代理并行处理，为每个板块生成手绘信息图
- 信息图风格与 optimize-article 技能保持一致

## 输入处理

根据 `$ARGUMENTS` 或用户上下文判断输入类型：

- **网页链接**（以 `http://` 或 `https://` 开头）：使用 `WebFetch` 工具抓取文章内容。
- **本地文件路径**（如 `./article.md`、`C:/路径/文件.md`）：使用 `Read` 工具读取文件内容。
- **直接粘贴内容**：用户直接在对话中粘贴了文章正文，以该内容作为来源。

若未提供任何参数，询问用户："请提供参考文章的链接、本地文件路径，或直接粘贴文章内容。"

---

## 工作流程

### 第零步 — 说明流程并确认

在执行任何操作之前，向用户说明本次任务的完整流程与目标：

> **即将执行：小红书信息图生成**
>
> **输入来源：** [网页链接 / 本地文件：路径 / 直接粘贴内容]
>
> **执行步骤：**
> 1. 获取文章内容
> 2. 按二级标题（`##`）分割板块，提炼每个板块的核心内容
> 3. 并行为每个板块生成手绘信息图
> 4. 汇总所有信息图，保存到输出目录
>
> **目标：** 将文章内容转化为适合小红书传播的视觉信息图，每个板块一张。
>
> 请确认是否按此流程执行，或告知需要调整的地方。

**等待用户确认后，方可进入第一步。**

---

### 第一步 — 获取内容与标题

根据输入类型使用对应工具获取完整文章内容，同时提取文章标题：

- **本地文件路径**：文章标题 = 文件名（去掉扩展名）。例如路径为 `D:\...\我用 Claude Code 1个月后.md`，标题即为 `我用 Claude Code 1个月后`。
- **网页链接**：文章标题 = 网页正文中的 `<h1>` 或文章首行标题。
- **直接粘贴内容**：文章标题 = 正文第一行（去掉 `#` 前缀）。

提取到的标题将在第三步中作为每张信息图的顶部大标题使用。

---

### 第二步 — 分割板块

1. 识别文章中所有以 `##` 开头的二级标题
2. 将文章按二级标题分割为多个独立板块
3. 若文章没有二级标题，则将整篇文章视为一个板块
4. 列出所有板块标题，向用户确认分割结果

**输出格式：**

```
已识别到以下板块：
1. [板块标题 1]
2. [板块标题 2]
...

共 N 个板块，将并行生成 N 张信息图。
```

---

### 第三步 — 并行生成信息图

**使用子代理并行处理所有板块，但每个子代理启动前须等待前一个启动后间隔 10 秒，避免并发请求过多导致 API 断连。**

每个子代理负责一个板块，执行以下子流程：

#### 子流程：单板块信息图生成

1. **提炼框架文字**

   阅读该板块全文，提炼出以下结构的文字描述（不超过 100 字）：
   ```
   主题：[该板块的核心主题，5字以内]
   核心观点：[2-4个关键点，每点 5-10 字]
   逻辑关系：[并列 / 递进 / 因果 / 对比 中选一]
   视觉重点：[最需要突出的 1 个词或概念]
   ```

2. **拼装生图提示词**

   提示词只包含板块内容本身的精炼总结，**禁止添加任何说明性前缀**（如"请为...板块生成"、"图片顶部显示..."等多余句子）。

   格式如下：
   ```
   板块：[板块标题]。[核心观点1]，[核心观点2]，[核心观点3]，[核心观点4]。逻辑：[并列/递进/因果/对比]。重点：[视觉重点词]。风格：手绘笔记，极简，9:16竖版，纯白背景，中文标注。
   ```

   示例：
   ```
   板块：安全防护。执行前先描述流程再确认，高风险操作必须二次确认，不确定时必须先问不要猜测，错误记录到项目记忆避免重犯。逻辑：并列。重点：执行前确认。风格：手绘笔记，极简，9:16竖版，纯白背景，中文标注。
   ```

3. **调用生图脚本**

   若非第一个板块，先等待 10 秒再执行：
   ```bash
   sleep 10  # 非第一个板块执行前等待，防止并发请求过多
   PYTHONIOENCODING=utf-8 python .claude/skills/xiaohongshu/scripts/infographic_generator.py "提示词" "IMAGE_DIR的绝对路径"
   ```

   脚本执行后会输出图片保存路径（格式：`File saved to to: <路径>`），从中提取完整路径。

4. **返回结果**

   子代理返回：板块标题 + 图片路径（或失败原因）

---

### 第四步 — 汇总结果

所有子代理完成后，主进程汇总结果：

1. **确定输出目录**

   根据文章来源确定 `IMAGE_DIR`，在 `draft/` 目录下创建以文章标题命名的文件夹及 `image/` 子目录：
   ```
   ARTICLE_DIR = D:\BaiduSyncdisk\claude_code\article_writing\draft\<规范化的文章标题>
   IMAGE_DIR = <ARTICLE_DIR>\image
   ```

   **目录名规范化规则：**
   - 去除 Windows 不允许的字符：`\ / : * ? " < > |`
   - **空格一律替换为下划线 `_`**
   - 保留中文、英文、数字、下划线、连字符
   - 长度不超过 50 个字符（超出部分截断）

2. **输出汇总报告**

   ```
   ## 小红书信息图生成完成

   共生成 N 张信息图，保存在：`<IMAGE_DIR>`

   | 板块 | 图片路径 | 状态 |
   |------|----------|------|
   | [板块 1] | <路径> | ✓ 成功 |
   | [板块 2] | <路径> | ✓ 成功 |
   | [板块 3] | — | ✗ 失败（原因） |
   ```

3. **失败处理**

   若某个板块的信息图生成失败，在汇总报告中注明失败原因，不阻断整体流程。

---

## 执行约束

- 每个板块只生成一张信息图，不重复生成
- 图片文件名含时间戳，不会覆盖已有图片
- 若脚本调用失败，跳过并在输出中注明，不阻断整体流程
- **并行间隔**：启动多个子代理时，每个子代理启动前等待 10 秒（使用 `sleep 10`），避免并发 API 请求过多导致断连
- **终端命令须事先说明**：每次执行 Bash 命令前，须向用户说明该命令的具体用途，不得静默执行
- **不确定即暂停汇报**：执行过程中若遇到任何不确定、没把握的情况，必须立即暂停并向用户汇报
