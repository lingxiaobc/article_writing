# 公众号文章优化技能

## 技能定位

本技能专为**微信公众号文章**的优化场景设计，优化目标聚焦于：

- **文章质量**：严谨、逻辑合理、语义通顺、适配场景
- **语言风格**：日记风格，融入特定性格特征（内敛但渴望表达、思维活跃但焦虑、喜欢反思、技术敏感、产品化思维）
- **可读性**：适合移动端碎片化阅读，段落简短，节奏轻快
- **传播力**：标题抓眼球，开头留住读者，结尾引导互动（点赞 / 在看 / 转发）
- **信息图配套**：每个大板块自动生成手绘风格信息图，提升内容视觉层次

> 若文章来源为非公众号场景（如技术博客、学术文章），优化策略仍适用，但传播导向建议可酌情忽略。
>
> **重要说明**：
> - 日记风格仅作为**语言风格参考**（语气、表达方式等）
> - 优化后的文章必须保持：严谨、逻辑合理、语义通顺
> - **不保留**日记中的啰嗦、重复、语序混乱等问题

## 输入处理

根据 `$ARGUMENTS` 或用户上下文判断输入类型：

- **网页链接**（以 `http://` 或 `https://` 开头）：使用 `WebFetch` 工具抓取文章内容。
- **本地文件路径**（如 `./article.md`、`C:/路径/文件.md`）：使用 `Read` 工具读取文件内容。
- **直接粘贴内容**：用户直接在对话中粘贴了文章正文，以该内容作为来源。

若未提供任何参数，询问用户："请提供需要优化的文章链接、本地文件路径，或直接粘贴文章内容。"

### 第一步 — 获取内容
根据输入类型使用对应工具获取完整文章内容。

### 第二步 — 观点分析与建议

**目的：** 在优化之前，用客观、公正、理性的角度分析文章中的观点，找出观点层面的问题，并给出针对性的优化建议。

**执行流程：**

1. **提炼核心观点**
   - 阅读全文，提炼出文章的核心观点（通常 1-3 个主要观点）
   - 识别支撑核心观点的子观点

2. **逐个观点分析**
   对每个观点进行以下分析：
   - **合理性**：观点本身是否有道理？是否符合常识或逻辑？
   - **逻辑性**：观点推导过程是否严谨？有无逻辑跳跃或循环论证？
   - **完整性**：观点是否有遗漏的方面或边界条件？
   - **客观性**：是否存在主观臆断、情绪化表述或绝对化断言？
   - **论据支撑**：观点是否有充分的论据（事实、数据、案例）支撑？

3. **指出潜在问题**
   针对每个观点，指出可能存在的问题，例如：
   - 主观性过强，缺乏客观依据
   - 论据不足，结论下得太仓促
   - 逻辑跳跃，中间环节缺失
   - 片面性，只看到一面而忽略另一面
   - 绝对化表述，缺乏审慎态度
   - 概念模糊，定义不清晰

4. **给出优化建议**
   针对指出的问题，给出具体的优化建议：
   - 建议补充的论据或案例
   - 建议调整的表述方式（如将绝对化改为审慎性）
   - 建议补充的论证环节
   - 建议引入的辩证视角
   - 建议调整的观点范围或限定条件

5. **输出观点分析报告**
   按以下格式输出：

```
## 观点分析报告

### 核心观点提炼
1. [观点 1]
2. [观点 2]
...

### 观点逐项分析

#### 观点 1：[观点内容]
**合理性：** [分析]
**逻辑性：** [分析]
**完整性：** [分析]
**客观性：** [分析]
**论据支撑：** [分析]

**潜在问题：**
- [问题 1]
- [问题 2]
...

**优化建议：**
- [建议 1]
- [建议 2]
...

---

[其余观点同上格式]

### 整体优化方向建议
基于以上分析，建议在后续优化中重点关注：
1. [优先级 1]
2. [优先级 2]
3. [优先级 3]
```

**等待用户确认：**

观点分析报告输出后，询问用户：

> 观点分析已完成。以上是对文章观点的分析与建议。
>
> 请确认：
> 1. 以上分析是否准确？有无需要补充或修正的地方？
> 2. 是否按以上建议进行后续优化？或你有其他优化方向？

#### 第五步附加流程 — 每个大板块后生成信息图

在输出完整文章时，每完成一个**大板块**（开篇 / 各主要章节 / 结尾），按以下子流程生成配套信息图：

**子流程：**

1. **确定图片输出目录（每篇文章只做一次，在第一张图生成前执行）**

   根据文章来源确定 `IMAGE_DIR`：

   - **本地文件**：在 `draft/` 目录下创建以文章标题命名的文件夹，在其下创建 `image/` 子目录
     ```
     ARTICLE_DIR = D:\BaiduSyncdisk\claude_code\article_writing\draft\<规范化的文章标题>
     IMAGE_DIR = <ARTICLE_DIR>/image
     ```
     例如文章标题为 "Claude Code 使用心得"，则：
     ```
     ARTICLE_DIR = D:\BaiduSyncdisk\claude_code\article_writing\draft\Claude Code 使用心得
     IMAGE_DIR = D:\BaiduSyncdisk\claude_code\article_writing\draft\Claude Code 使用心得\image
     ```

     **目录名规范化规则：**
     - 去除 Windows 不允许的字符：`\ / : * ? " < > |`
     - 保留空格、中文、英文、数字、下划线、连字符
     - 长度不超过 50 个字符（超出部分截断）

   - **网页链接 / 直接粘贴**：同样在 `draft/` 目录下创建以文章标题命名的文件夹及 `image/` 子目录

2. **识别大板块边界**
   文章中以 `##` 或明显段落主题标记的独立章节视为一个大板块。

3. **提炼框架文字（生图前必做）**
   阅读该板块全文，提炼出以下结构的文字描述（不超过 100 字）：
   ```
   主题：[该板块的核心主题，5字以内]
   核心观点：[2-4个关键点，每点 5-10 字]
   逻辑关系：[并列 / 递进 / 因果 / 对比 中选一]
   视觉重点：[最需要突出的 1 个词或概念]
   ```

4. **拼装生图提示词**
   将上述框架转化为以下格式的提示词字符串：
   ```
   请为公众号文章的"[主题]"板块生成一张手绘信息图。
   核心内容：[核心观点逐条列出，逗号分隔]。
   逻辑关系：[逻辑关系类型]。
   视觉重点突出"[视觉重点]"。
   风格：手绘笔记，极简，16:9，纯白背景，中文标注。
   ```

5. **调用生图脚本**

   无论来源是什么，统一传入 `IMAGE_DIR` 参数：
   ```bash
   python .claude/skills/optimize-article/scripts/infographic_generator.py "提示词" "IMAGE_DIR的绝对路径"
   ```

   脚本执行后会输出图片保存路径（格式：`File saved to to: <路径>`），从中提取完整路径。

6. **将图片嵌入文章**
   在该板块文字内容结束后，紧接着插入：
   ```markdown
   ![板块信息图](<图片绝对路径>)
   ```

> **注意：**
> - 每个大板块只生成一张信息图，不重复生成
> - 若脚本调用失败，跳过并在输出中注明"（信息图生成失败，请手动重试）"，不阻断整体流程
> - 图片文件名含时间戳，不会覆盖已有图片

### 第六步 — 保存文件

**执行流程：**

1. **创建文章专属目录**

   在 `D:\BaiduSyncdisk\claude_code\article_writing\draft\` 下创建以文章标题命名的目录：

   ```bash

   **目录名规范化规则：**
   - 去除 Windows 不允许的字符：`\ / : * ? " < > |`
   - 保留空格、中文、英文、数字、下划线、连字符
   - 长度不超过 50 个字符（超出部分截断）

2. **保存优化后的文章**

   将优化后的文章保存到新创建的目录中，文件名为 `<规范化的文章标题>.md`：

   ```

3. **告知用户保存结果**

   > 文章已保存到：`D:\BaiduSyncdisk\claude_code\article_writing\draft\<规范化的文章标题>\<规范化的文章标题>.md`
   >
   > 配图已保存到：`D:\BaiduSyncdisk\claude_code\article_writing\draft\<规范化的文章标题>\image\`

> **注意：**
> - 若目标目录已存在，询问用户是否覆盖，不得静默覆盖
> - 若原文件来源为本地文件，不删除原文件，仅在新目录中创建优化版本

### 第七步 — 确认最终标题

文件保存完成后，询问用户是否需要调整标题：

**用户确认需要调整标题后，执行以下操作：**

2. 重命名文章目录：

4. 更新文章内容中的标题行（`# 标题`）
5. 告知用户最终文件路径

> **注意：**
> - 文件名和目录名中若含有 Windows 不允许的字符（`\ / : * ? " < > |`），自动去除
> - 若目标目录或文件已存在，询问用户是否覆盖，不得静默覆盖

### 第八步 — 生成朋友圈文案

标题确认完成后，自动生成朋友圈推广文案。

**执行流程：**

1. **提取文章核心信息**
   - 文章标题
   - 核心观点（1-3个）
   - 最吸引人的亮点或金句
   - 目标读者痛点

2. **调用文案生成脚本**
   ```bash
   python .claude/skills/optimize-article/scripts/moments_copy_generator.py "<文章文件路径>"
   ```

3. **文案生成原则**
   - 字数控制在 180-220 字
   - 开头用钩子吸引注意（提问 / 痛点 / 反常识）
   - 中间点出核心价值（解决什么问题 / 带来什么启发）
   - 结尾引导点击（悬念 / 行动召唤）
   - 语气口语化，符合朋友圈场景
   - 避免标题党和过度营销感

4. **输出格式**
   ```
   ## 朋友圈文案

   [文案正文，180-220字]

   ---

   **话题标签建议：** #标签1 #标签2 #标签3

   **配图建议：** 使用文章中的信息图，或文章封面图
   ```

5. **保存文案**
   将生成的文案追加到文章文件末尾，或单独保存为 `<文章标题>_朋友圈文案.txt`

6. **告知用户**
   > 朋友圈文案已生成，保存在文章末尾。你可以直接复制使用，或根据需要调整。

> **注意：**
> - 文案生成失败不阻断整体流程，提示用户手动编写
> - 文案仅供参考，用户可根据实际情况调整

## 执行约束

- 保留原作者的写作风格与核心观点，不改变文章的基本立场。
- 不捏造事实、数据或引用来源。
- 文章为中文则输出中文，英文则输出英文，除非用户明确要求切换语言。
- 修改以必要为原则，不做无意义的大幅改写。
- **文章质量要求**：确保优化后的文章严谨、逻辑合理、语义通顺、适配场景
- **风格融入**：自然融入日记风格的语言特征，不保留啰嗦、重复、语序混乱等问题
- 对不确定原意的段落，标注说明而非擅自猜测修改。
- **排版间距**：输出文章时，图片（`![](...)`）和标题（`##`、`###` 等）的前后各保留一个空行，保持视觉呼吸感。
- **终端命令须事先说明**：每次执行 Bash 命令前，须向用户说明该命令的具体用途，不得静默执行。
- **不确定即暂停汇报**：执行过程中若遇到任何不确定、没把握的情况，必须立即暂停并向用户汇报，等待指示后再继续。

## 长文档写入规范

**适用场景**：当优化后的文章预计**超过 200 行**时，必须遵循以下流程。

### 标准流程

1. **第一步：编写框架**
   - 先构思完整的文章框架（开头、各板块标题、结尾）
   - 将框架写入文件，包含：
     - 文章标题
     - 各板块的二级标题（`##`）
     - 各板块的核心要点（简要列出）
   - 告知用户："框架已完成，请审核。确认无误后将逐板块填充内容。"
   - 等待用户确认

2. **第二步：逐板块填充内容**
   - 按框架顺序，逐个板块思考并填充内容
   - 每完成一个板块，立即使用 `Edit` 工具写入文件
   - 不要一次性思考所有板块，避免"手没跟上脑子"
   - 每完成 2-3 个板块，向用户汇报进度

3. **第三步：全文检查**
   - 所有板块填充完成后，**必须**通读全文
   - 检查板块之间的衔接是否自然
   - 检查逻辑是否合理、前后是否一致
   - 检查是否有遗漏的信息图
   - 发现问题立即修正

### 子代理并行处理

**适用场景**：当文章中有多个独立板块（板块之间关联性不强）时，可使用子代理并行处理。

**操作方法**：
- 将独立板块分配给不同的子代理
- 每个子代理独立完成自己的板块内容
- 所有子代理完成后，主进程汇总结果
- 汇总后**必须**进行全文检查，确保板块之间衔接合理

**注意事项**：
- 开头和结尾不适合用子代理，因为它们需要与全文呼应
- 如果板块之间有强逻辑关系（如递进、因果），不适合并行处理
