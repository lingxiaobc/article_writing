# 公众号文章优化技能

## 技能定位

本技能专为**微信公众号文章**的优化场景设计，优化目标聚焦于：

- **文章质量**：严谨、逻辑合理、语义通顺、适配场景
- **语言风格**：日记风格，融入特定性格特征（内敛但渴望表达、思维活跃但焦虑、喜欢反思、技术敏感、产品化思维）
- **可读性**：适合移动端碎片化阅读，段落简短，节奏轻快
- **传播力**：标题抓眼球，开头留住读者，结尾引导互动（点赞 / 在看 / 转发）
- **信息图配套**：每个大板块自动生成手绘风格信息图，提升内容视觉层次

> 若文章来源为非公众号场景（如技术博客、学术文章），优化策略仍适用，但传播导向建议可酌情忽略。
>
> **重要说明**：
> - 日记风格仅作为**语言风格参考**（语气、表达方式等）
> - 优化后的文章必须保持：严谨、逻辑合理、语义通顺
> - **不保留**日记中的啰嗦、重复、语序混乱等问题

## 输入处理

根据 `$ARGUMENTS` 或用户上下文判断输入类型：

- **网页链接**（以 `http://` 或 `https://` 开头）：使用 `WebFetch` 工具抓取文章内容。
- **本地文件路径**（如 `./article.md`、`C:/路径/文件.md`）：使用 `Read` 工具读取文件内容。
- **直接粘贴内容**：用户直接在对话中粘贴了文章正文，以该内容作为来源。

若未提供任何参数，询问用户："请提供需要优化的文章链接、本地文件路径，或直接粘贴文章内容。"

## 优化工作流程

按以下步骤依次执行：

### 第零步 — 说明流程并确认

在执行任何操作之前，向用户说明本次任务的完整流程与目标，格式如下：

> **即将执行：公众号文章优化**
>
> **输入来源：** [网页链接 / 本地文件：路径 / 直接粘贴内容]
>
> **执行步骤：**
> 1. 获取文章内容
> 2. 调用 article-critic 子智能体进行批判性评审，并逐项核查写作清单
> 3. 按优化原则对文章进行全面优化（保持严谨、逻辑合理、语义通顺）
> 4. 融入日记风格的语言特征（语气、表达方式）
> 5. 输出诊断评分表、改动说明及完整优化版文章（每个大板块后附手绘信息图）
> 6. 询问是否保存文件
>
> **目标：** 提升文章质量（严谨、逻辑合理、语义通顺）的同时，融入日记风格的语言特征，提升可读性、传播力与视觉层次。
>
> 请确认是否按此流程执行，或告知需要调整的地方。

**等待用户确认后，方可进入第一步。**

---

### 第一步 — 获取内容
根据输入类型使用对应工具获取完整文章内容。

### 第二步 — 分析诊断
调用 `article-critic` 子智能体（见 `../../agents/article-critic.md`）对文章进行批判性评审。
评审完成后，参照 [references/diary_style_checklist.md](references/diary_style_checklist.md) 逐项核查，确保文章风格符合要求。

### 第三步 — 应用优化原则
按照 [principles.md](principles.md) 中定义的优化原则对文章进行优化。

> **说明：**
> - `principles.md` 是专门存放文章优化规则的文件
> - 优化时必须确保：严谨、逻辑合理、语义通顺、适配场景
> - 日记风格（`style_guide.md`）仅作为语言风格参考，不保留啰嗦、重复、语序混乱等问题
> - 若该文件尚不存在，则按通用写作最佳实践执行，并提示用户将具体优化原则补充到 `principles.md` 中

若需分析文章字数与可读性，可运行：
```bash
python .claude/skills/optimize-article/scripts/word_count.py <文件路径>
python .claude/skills/optimize-article/scripts/extract_outline.py <文件路径>
```

### 第四步 — 输出结果
严格按照 [assets/output_template.md](assets/output_template.md) 定义的格式交付优化结果，包含：
- 诊断评分表
- 主要改动说明（逐条列出）
- 完整优化版文章（含每个大板块后的信息图，见下方第四步附加流程）

#### 第四步附加流程 — 每个大板块后生成信息图

在输出完整文章时，每完成一个**大板块**（开篇 / 各主要章节 / 结尾），按以下子流程生成配套信息图：

**子流程：**

1. **确定图片输出目录（每篇文章只做一次，在第一张图生成前执行）**

   根据文章来源确定 `IMAGE_DIR`：

   - **本地文件**：取文件所在目录，在其下创建 `image/` 子目录
     ```
     IMAGE_DIR = <文章文件所在目录>/image
     ```
     例如文章路径为 `P:\claude_code\article_writing\draft\我用扣子搭建了一个信息图生成网页\xxx.md`，
     则 `IMAGE_DIR = P:\claude_code\article_writing\draft\我用扣子搭建了一个信息图生成网页\image`

   - **网页链接 / 直接粘贴**：使用默认目录，脚本自动处理，调用时不传第二个参数

2. **识别大板块边界**
   文章中以 `##` 或明显段落主题标记的独立章节视为一个大板块。

3. **提炼框架文字（生图前必做）**
   阅读该板块全文，提炼出以下结构的文字描述（不超过 100 字）：
   ```
   主题：[该板块的核心主题，5字以内]
   核心观点：[2-4个关键点，每点 5-10 字]
   逻辑关系：[并列 / 递进 / 因果 / 对比 中选一]
   视觉重点：[最需要突出的 1 个词或概念]
   ```

4. **拼装生图提示词**
   将上述框架转化为以下格式的提示词字符串：
   ```
   请为公众号文章的"[主题]"板块生成一张手绘信息图。
   核心内容：[核心观点逐条列出，逗号分隔]。
   逻辑关系：[逻辑关系类型]。
   视觉重点突出"[视觉重点]"。
   风格：手绘笔记，极简，16:9，纯白背景，中文标注。
   ```

5. **调用生图脚本**

   - 来源为本地文件时（传入 `IMAGE_DIR`）：
     ```bash
     python .claude/skills/optimize-article/scripts/infographic_generator.py "提示词" "IMAGE_DIR的绝对路径"
     ```
   - 来源为网页或粘贴内容时（不传目录，使用默认）：
     ```bash
     python .claude/skills/optimize-article/scripts/infographic_generator.py "提示词"
     ```

   脚本执行后会输出图片保存路径（格式：`File saved to to: <路径>`），从中提取完整路径。

6. **将图片嵌入文章**
   在该板块文字内容结束后，紧接着插入：
   ```markdown
   ![板块信息图](<图片绝对路径>)
   ```

> **注意：**
> - 每个大板块只生成一张信息图，不重复生成
> - 若脚本调用失败，跳过并在输出中注明"（信息图生成失败，请手动重试）"，不阻断整体流程
> - 图片文件名含时间戳，不会覆盖已有图片

### 第五步 — 保存文件（如适用）
若来源为本地文件，询问用户：
"是否将优化后的内容覆盖原文件，还是另存为新文件（如 `article-optimized.md`）？"

## 执行约束

- 保留原作者的写作风格与核心观点，不改变文章的基本立场。
- 不捏造事实、数据或引用来源。
- 文章为中文则输出中文，英文则输出英文，除非用户明确要求切换语言。
- 修改以必要为原则，不做无意义的大幅改写。
- **文章质量要求**：确保优化后的文章严谨、逻辑合理、语义通顺、适配场景
- **风格融入**：自然融入日记风格的语言特征，不保留啰嗦、重复、语序混乱等问题
- 对不确定原意的段落，标注说明而非擅自猜测修改。
- **排版间距**：输出文章时，图片（`![](...)`）和标题（`##`、`###` 等）的前后各保留一个空行，保持视觉呼吸感。
- **终端命令须事先说明**：每次执行 Bash 命令前，须向用户说明该命令的具体用途，不得静默执行。
- **不确定即暂停汇报**：执行过程中若遇到任何不确定、没把握的情况，必须立即暂停并向用户汇报，等待指示后再继续。
